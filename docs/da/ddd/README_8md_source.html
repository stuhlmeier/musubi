3 Simple application framework in modern C++. Written from scratch for maximum usability.

5 ## Building
6 
7 ```sh
8 mkdir build
9 cd build
10 cmake .. && make all install
11 ```
12 
13 This will install `libmusubi` to /usr/local by default.
14 
15 musubi requires C++17 features and support for older standards is not a goal.
16 It will likely make extensive use of C++20 (concepts, ranges, `std::format`, `std::numbers`, `consteval`, ...)
17 once the standard is published in the near future.
18 
19 Currently, the library compiles and the sample runs fine on both Clang and GCC.
20 
21 **Note:** The CMake scripts for this project assume SDL2 2.0.8 (March 2018) or later is installed.
22 Earlier versions of SDL [do not configure a `SDL2::SDL2` target][arch-sdlcmake], which musubi tries to import.
23 
24 Particularly of note is that Ubuntu's current version of `libsdl2-dev` (as of eoan)
25 suffers from this issue; building/installing SDL2 from source is required here.
26 Rolling-release distributions (such as Arch or Gentoo) are up-to-date and therefore unaffected.

28 ## Components
29 
30 ### `libmusubi`
31 
32 `libmusubi` is the main "engine" component and currently consists of:
33 
34  - graphics abstractions
35  - windowing (SDL2)
36  - pixmaps
37  - rendering
38  - shapes (OpenGL)
39  - textures (OpenGL)
40  - lifecycle abstractions
41  - an `application` owns `window`s, which own `screen`s
42  - asset packing & managing
43  - assets are automatically packed in `xz` archives by `mpack.py`, and can be loaded and read at runtime
44 
45 Currently, `libmusubi` will assume and request OpenGL 3.3 or greater by default.
46 No abstractions for versions older than 3.0 are planned, due to API differences.
47 
48 Once the default API is stable enough, all OpenGL abstractions will be given an abstract base type
49 so they will be easy to swap out with other implementations (i.e. other OpenGL versions,
50 possibly even other rendering APIs).
