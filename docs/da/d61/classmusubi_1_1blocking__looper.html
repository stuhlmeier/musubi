Detailed Description

An event looper that loops infinitely on the calling thread.

This is best suited for applications handling input on the main thread (i.e. most applications.)

Calling loop() will block forever, so this is typically called after initial setup. To stop the looper and the application, request_stop can be invoked on an event looper, or on one of its stop_tokens.

Loopers are constructed with a specified tick function, which is executed on every iteration; actions can be added to be executed after the tick function.
